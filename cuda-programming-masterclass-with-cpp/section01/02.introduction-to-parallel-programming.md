# 🚀 CUDA 마스터클래스: 병렬 프로그래밍의 기초

## 🖥️ 컴퓨터 아키텍처와 프로세스의 이해

### 폰 노이만 아키텍처의 기본 원리

현대 컴퓨터의 근간이 되는 폰 노이만 아키텍처는 프로그램 실행의 기본 원리를 정의한다. 프로그램이 실행될 때:

1. 실행 파일은 하드 드라이브에 저장됨
2. 실행 시점에서:
   - 명령어는 CPU로 로드
   - 기본 데이터는 RAM으로 로드
3. 프로세서가 입력을 받아 명령어를 실행하고 결과 출력
4. 이 과정은 프로그램의 복잡도와 관계없이 동일하게 적용

### 🔄 프로세스와 멀티태스킹의 마법

현대의 컴퓨터에서는 Visual Studio, Chrome, Word 등 여러 프로그램을 동시에 실행할 수 있습니다. 이것이 가능한 이유는 프로세스라는 개념 때문입니다.

**프로세스의 구성 요소:**

- 실행 코드 이미지
- RAM의 메모리 할당
- 컨텍스트 정보
  - 프로세스 상태 데이터
  - 메모리 주소
  - 프로그램 카운터 상태
  - 기타 실행 관련 정보

### ⚡ 컨텍스트 스위칭의 동작 원리

단일 코어 프로세서에서도 여러 프로그램을 "동시에" 실행할 수 있는 비결은 컨텍스트 스위칭에 있습니다.

**작동 방식:**

1. 라운드 로빈 방식으로 프로세스 실행
2. 각 프로세스에 시간 슬롯 할당
3. 프로세스 우선순위와 준비 상태에 따라 실행 시간 조절
4. 프로세스 전환 시 컨텍스트 저장 및 복원

## 🧵 스레드와 병렬 처리의 세계

### 스레드: 프로세스의 작은 실행 단위

스레드는 독립적으로 관리될 수 있는 가장 작은 프로그램 실행 단위입니다.

**스레드의 특징:**

- 프로세스의 구성 요소
- 모든 프로세스는 메인 스레드 보유
- 프로세스 내 여러 스레드가 메모리 공유
- "경량 프로세스"라고도 불림

### 🏗️ 병렬 처리의 실제 예: 주택 건설 프로젝트

주택 건설을 통해 병렬 처리의 개념을 쉽게 이해할 수 있습니다.

**순차적으로 진행해야 하는 작업:**

- 건축 계획 수립
- 기초 공사
- 구조물 검사

**병렬로 진행 가능한 작업:**

- 벽돌 쌓기
- 배관 공사
- 전기 배선
- 하수관 설치
- 내부 마감

## 🔄 병렬성과 동시성의 차이

### 진정한 병렬성(True Parallelism)

멀티코어 환경에서 실제로 여러 작업이 동시에 실행되는 것을 의미합니다.

**특징:**

- 물리적으로 독립된 코어에서 실행
- 실제 동시 처리
- 성능 향상이 선형적으로 발생 가능

### 동시성(Concurrency)

단일 코어에서 컨텍스트 스위칭을 통해 구현되는 "가상의" 병렬 처리입니다.

**특징:**

- 빠른 전환으로 동시 실행처럼 보임
- 실제로는 순차 실행
- 오버헤드 존재

## 🎯 병렬 프로그래밍의 두 가지 접근 방식

### 1. 작업 수준 병렬성(Task-Level Parallelism)

서로 다른 작업을 동시에 실행하는 방식입니다.

**예시:**

- 주택 건설에서 전기 작업자와 배관공이 동시에 작업
- 웹 서버에서 여러 요청을 동시에 처리
- 멀티미디어 응용프로그램에서 비디오와 오디오 동시 처리

### 2. 데이터 수준 병렬성(Data-Level Parallelism)

동일한 작업을 여러 데이터에 동시에 적용하는 방식입니다.

**예시:**

- 여러 작업자가 동시에 벽돌 쌓기
- 이미지 처리에서 각 픽셀에 필터 적용
- 대규모 데이터 정렬

## 💡 결론

현대의 프로그래밍에서 병렬 처리는 선택이 아닌 필수가 되었습니다. CUDA를 통한 GPU 프로그래밍은 대규모 병렬 처리의 강력한 도구이며, 이러한 기본 개념들의 이해는 효율적인 CUDA 프로그래밍의 기초가 됩니다.
